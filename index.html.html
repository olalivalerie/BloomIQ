Features

  • Monitors soil moisture, temperature and humidity in real-time
  • Automatic pump control with configurable thresholds or ML-based decisions
  • Works for farms, home gardens, office plants and flower beds
  • Offline-capable demo dashboard for presentations
  • Includes synthetic dataset for model prototyping

How BloomIQ works

  1. Sensors (soil moisture probe, DHT11/22 for temp/humidity, optional light sensor) collect environmental data.
  2. An ESP32 (or similar microcontroller) reads sensors and applies watering logic (rule-based or ML model).
  3. A relay switches a water pump or solenoid valve to irrigate when needed.
  4. Optional GSM module can send SMS alerts; data can be logged locally or sent to a cloud/dashboard.

Training Dataset (synthetic)

For model prototyping, use the provided synthetic CSV with readings and heuristic labels.

Download bloomiq_dataset.csv

Dataset description

Columns: timestamp, soil_moisture_pct, temperature_c, humidity_pct, light_lux, rainfall_mm, pump, state_label

state_label values include: healthy, dry, overwatered, heat_stressed.

How to use

  • Save this file as index.html. Place bloomiq_dataset.csv in the same folder to enable the download link.
  • Double-click index.html to open in a browser (works offline).
  • To publish online later, upload these files to a GitHub repository and enable GitHub Pages.
